## 工作原理

### 交易时间（9:30-11:30, 13:00-15:00）
```
前端请求 (每 3 秒) 
    ↓
后端 API 接口
    ↓
调用数据源 API (腾讯/新浪/东方财富)
    ↓
返回最新数据
    ↓
前端更新显示
```

### 非交易时间
```
前端请求
    ↓
后端 API 接口
    ↓
返回缓存数据
    ↓
前端显示
```

## 修改内容

### server.js 优化

**修改前：**
```javascript
// 定时任务每分钟更新
cron.schedule('* * * * *', async () => {
  if (isTradingTime()) {
    await fetchAllData();
  }
});

// API 直接返回缓存
app.get('/api/all', (req, res) => {
  res.json(marketData);
});
```

**修改后：**
```javascript
// 只在收盘时保存缓存
cron.schedule('* * * * *', async () => {
  if (isCloseToClose()) {
    saveCache();
  }
});

// API 实时获取数据
app.get('/api/all', async (req, res) => {
  if (isTradingTime()) {
    await fetchAllData();
  }
  res.json(marketData);
});
```

## 性能优化

### 数据更新策略

1. **交易时间**：
   - 每次 API 请求都获取最新数据
   - 确保数据实时性
   - 延迟：1-3 秒（API 响应时间）

2. **非交易时间**：
   - 使用缓存数据
   - 快速响应
   - 减少 API 调用

3. **缓存管理**：
   - 每日收盘后自动保存
   - 按日期存储
   - 自动清理过期缓存

## 注意事项

1. **刷新频率**：建议 3 秒或以上，避免 API 限流
2. **交易时间**：只有交易时间数据才会变化
3. **网络延迟**：数据更新速度取决于网络状况
4. **API 稳定性**：已配置备用数据源
