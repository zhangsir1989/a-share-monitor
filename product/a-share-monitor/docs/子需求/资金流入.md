# 对接 MyData 逐笔成交明细计算主力资金流入流出设计文档

# 1. 文档概述

## 1.1 文档目的

本文档旨在明确对接 MyData 接口、通过逐笔成交明细数据计算主力资金流入、流出及净流入的完整实现方案。

## 1.2 核心目标

- 精准对接 MyData 相关 API，稳定获取单只股票的逐笔成交、股票基础信息等核心数据
- 通过动态阈值（基于总流通市值）实现大小单精准分类，区分散户、中户、主力资金
- 严格按照交易所标准，计算总资金及主力资金的流入、流出、净流入，剔除集合竞价等无效数据
- 处理数据异常、接口异常，所有异常场景均输出标准化日志

## 1.3 前置约定

- MyData 接口已申请授权（拥有有效 licence），可正常调用逐笔成交、股票基础信息接口
- 核心数据依赖：单只股票的逐笔成交明细（含时间、成交价、成交量、方向）、流通股本、最新股价
- 大小单分类采用"总流通市值动态阈值"，统一基于总流通市值计算
- 主力资金定义：大单 + 特大单对应的主动成交资金，散户资金定义：小单对应的主动成交资金，中单暂不纳入主力/散户分类
- 所有异常场景（接口调用失败、数据为空、数据异常等）均输出标准化日志

# 2. 需求分析

## 2.1 功能需求

### 2.1.1 数据对接功能

- 对接 MyData 逐笔成交接口（zbjy），获取指定单只股票的逐笔成交明细
- 对接 MyData 股票基础信息接口（instrument），获取单只股票的流通股本、最新股价等基础数据
- 支持指定交易日的数据筛选，仅支持单只股票查询

### 2.1.2 数据清洗功能

- 剔除集合竞价数据：过滤时间为 9:25:00 或方向为 0（中性盘）且属于集合竞价的成交记录
- 异常数据过滤：剔除成交价为 0、成交量为 0 的无效数据
- 数据格式标准化：统一成交金额、成交量的单位

### 2.1.3 大小单分类功能

- 计算流通市值：结合单只股票的流通股本、最新股价计算流通市值
- 动态阈值计算：基于流通市值设定小单、中单、大单、特大单的金额阈值
- 逐笔分类：根据每笔成交金额，匹配对应阈值，标记每笔成交的类型

### 2.1.4 资金流向计算功能

- 总资金流向计算：统计所有主动买入（方向=1/B）、主动卖出（方向=2/S）的成交金额
- 主力资金流向计算：仅统计大单、特大单的主动买入、主动卖出金额
- 散户资金流向计算：统计小单的主动买入、主动卖出金额

### 2.1.5 结果输出功能

- 输出核心计算结果：总资金流入/流出/净流入、主力资金流入/流出/净流入、散户资金流入/流出/净流入
- 输出辅助信息：流通市值、各档位阈值、逐笔分类明细、异常数据统计
- 支持 JSON 格式输出

## 2.2 非功能需求

- 稳定性：接口调用失败时支持重试机制（最多 3 次）
- 精准性：大小单分类误差≤5%，资金流向计算符合交易所标准
- 效率：单只股票单日逐笔数据计算耗时≤30 秒
- 可排查性：所有异常场景均输出标准化日志

# 3. MyData API 接口

## 3.1 股票基础信息接口

**接口地址**：`https://api.mairuiapi.com/hsstock/instrument/{股票代码}.{市场}/{licence}`

**示例**：`https://api.mairuiapi.com/hsstock/instrument/000001.SZ/FB1A859B-6832-4F70-AAA2-38274F23FC90`

**返回字段**：
- `ii`: 股票代码
- `ei`: 交易所 (SH/SZ)
- `name`: 股票名称
- `pc`: 昨收价（元）
- `fv`: 流通股本（股）
- `tv`: 总股本（股）
- `up`: 涨停价
- `dp`: 跌停价

## 3.2 逐笔成交接口

**接口地址**：`https://api.mairuiapi.com/hsrl/zbjy/{股票代码}/{licence}`

**示例**：`https://api.mairuiapi.com/hsrl/zbjy/000001/FB1A859B-6832-4F70-AAA2-38274F23FC90`

**返回字段**：
- `d`: 日期
- `t`: 时间（HH:mm:ss）
- `v`: 成交量（股）
- `p`: 成交价（元）
- `ts`: 方向（0=中性，1=买入，2=卖出）

# 4. 计算逻辑

## 4.1 流通市值计算

```
流通市值 = 流通股本 × 最新股价
```

## 4.2 动态阈值计算

基于流通市值，设定以下金额阈值：

- **小单阈值**：流通市值 × 0.001%（即 0.00001）
- **中单阈值**：流通市值 × 0.005%（即 0.00005）
- **大单阈值**：流通市值 × 0.02%（即 0.0002）
- **特大单阈值**：流通市值 × 0.05%（即 0.0005）

## 4.3 逐笔分类逻辑

对每笔清洗后的逐笔成交，根据成交金额匹配阈值：

- **小单**：成交金额 ＜ 小单阈值
- **中单**：小单阈值 ≤ 成交金额 ＜ 中单阈值
- **大单**：中单阈值 ≤ 成交金额 ＜ 大单阈值
- **特大单**：成交金额 ≥ 大单阈值

## 4.4 资金流向计算

### 基础定义

- **主动买入金额**：方向 ts=1（主动买）的成交金额，计入"流入"
- **主动卖出金额**：方向 ts=2（主动卖）的成交金额，计入"流出"
- **净流入** = 流入 - 流出（正数为净流入，负数为净流出）

### 总资金流向计算

- 总流入 = 所有主动买入金额之和（不分大小单）
- 总流出 = 所有主动卖出金额之和（不分大小单）
- 总净流入 = 总流入 - 总流出

### 主力资金流向计算（核心）

- 主力流入 = 大单主动买入金额 + 特大单主动买入金额
- 主力流出 = 大单主动卖出金额 + 特大单主动卖出金额
- 主力净流入 = 主力流入 - 主力流出
- 主力资金占比 = （主力流入 + 主力流出）/ （总流入 + 总流出）

### 散户资金流向计算

- 散户流入 = 小单主动买入金额
- 散户流出 = 小单主动卖出金额
- 散户净流入 = 散户流入 - 散户流出

# 5. 输出格式

## 5.1 基础信息

- 股票代码、股票名称、市场
- 流通股本、流通市值（亿元）
- 最新股价

## 5.2 阈值信息（万元）

- 小单阈值、中单阈值、大单阈值、特大单阈值

## 5.3 统计信息

- 原始逐笔数据量、清洗后有效数据量、剔除的异常数据量

## 5.4 核心计算结果（亿元）

| 资金类型 | 流入 | 流出 | 净流入 |
|---------|------|------|--------|
| 总资金 | 总流入金额 | 总流出金额 | 总净流入金额 |
| 特大单 | 特大单流入 | 特大单流出 | 特大单净流入 |
| 大单 | 大单流入 | 大单流出 | 大单净流入 |
| 中单 | 中单流入 | 中单流出 | 中单净流入 |
| 小单 | 小单流入 | 小单流出 | 小单净流入 |
| 主力（大单 + 特大单） | 主力流入 | 主力流出 | 主力净流入 |

# 6. 异常处理

## 6.1 接口异常处理

- 接口调用失败：响应码非 200、超时、连接失败，触发重试机制（最多 3 次）
- 数据为空：接口返回空数据，输出日志并终止流程
- 接口限流：MyData 返回限流提示，暂停 5-10 秒后重试

## 6.2 数据异常处理

- 流通股本为 0：输出日志，终止计算
- 股价为 0：输出日志，终止计算
- 逐笔数据异常：成交价、成交量异常，直接剔除，不影响其他数据计算
- 方向数据异常：ts 字段非 0、1、2，剔除该笔数据

## 6.3 计算异常处理

- 资金流向计算为 NaN：所有逐笔数据均被剔除，输出流入、流出、净流入均为 0
- 阈值计算异常：流通市值为 0，跳过该股票计算

# 7. 配置参数

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| MyData licence | FB1A859B-6832-4F70-AAA2-38274F23FC90 | 用户申请的 MyData 授权码 |
| 接口重试次数 | 3 次 | 接口调用失败后的重试次数 |
| 重试间隔 | 1 秒 | 每次重试的时间间隔 |
| 动态阈值比例 | 小单 0.001%、中单 0.005%、大单 0.02%、特大单 0.05% | 基于流通市值的阈值比例 |
| 输出格式 | JSON | 支持 JSON 格式输出 |

- 总流通市值计算准确
- 大小单分类符合阈值规则
- 资金流向计算偏差≤10%（与专业软件对比）
