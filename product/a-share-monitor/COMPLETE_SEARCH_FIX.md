## 问题分析

### 根本原因
1. **拼音格式混乱** - 231 只股票拼音带空格（如"ZS RJ"）
2. **搜索逻辑单一** - 只支持完全匹配，不支持前缀/包含匹配
3. **大小写敏感** - 输入小写无法匹配大写拼音

---

## 修复方案

### 修复 1：统一拼音格式 ✅

**修复内容**:
- 去除所有拼音中的空格
- 统一转换为大写
- 修复 231 只股票

**示例**:
```javascript
// 修复前
{ name: '深证软件', pinyin: 'ZS RJ' }

// 修复后
{ name: '深证软件', pinyin: 'ZSRJ' }
```

---

### 修复 2：优化搜索逻辑 ✅

**多级匹配策略**:
```javascript
// 1. 代码匹配（支持 sh/sz 前缀）
if (code.match(/(SH|SZ)?(\d{6})/i)) {
  // 直接查询
}

// 2. 名称包含匹配（中文搜索）
if (stock.name.includes(query)) {
  return true;
}

// 3. 拼音完全匹配
if (stock.pinyin === query) {
  return true;
}

// 4. 拼音前缀匹配（如 SHZ → SHZS）
if (stock.pinyin.startsWith(query) && query.length >= 2) {
  return true;
}

// 5. 拼音包含匹配（如 ZS → 所有 ZS 开头）
if (stock.pinyin.includes(query) && query.length >= 2) {
  return true;
}
```

---

### 修复 3：大小写不敏感 ✅

**统一转换**:
```javascript
const queryUpper = query.toUpperCase();

// 所有比较都使用大写
if (stock.pinyin === queryUpper) { }
if (stock.pinyin.startsWith(queryUpper)) { }
```

## 技术细节

### 正则表达式
```javascript
// 匹配代码（支持 sh/sz 前缀，大小写不敏感）
/(SH|SZ)?(\d{6})/i
```

### 匹配优先级
1. 完全匹配（代码/名称/拼音）
2. 前缀匹配（拼音/代码）
3. 包含匹配（拼音/代码）

### 排序规则
1. 完全匹配优先
2. 前缀匹配次之
3. 按代码升序排列
