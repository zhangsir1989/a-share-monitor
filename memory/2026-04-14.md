## 2026-04-14 01:15 UTC - 修复自选股页面删除股票时使用错误 type 值的问题

### 用户反馈

**问题**：在"机器人"分组点击删除按钮，前端显示"已移除"，但数据库数据未删除，刷新页面后股票仍然存在。

**截图证据**：
- 用户在"机器人"分组（type=3）有 3 只股票
- 点击删除按钮后股票仍在列表中

### 问题分析

#### 根本原因

**custom.js deleteStockFromDb()**：
```javascript
async function deleteStockFromDb(code, market) {
  const response = await fetch('/api/custom-stocks/delete', {
    body: JSON.stringify({ 
      stock_code: code, 
      stock_market: market, 
      type: 1  // ❌ 硬编码 type=1
    })
  });
}
```

**问题**：硬编码 `type: 1`，但用户在 type=3 分组删除股票！

#### 问题流程

```
1. 用户在"机器人"分组 (type=3) 点击删除 159530
   ↓
2. 前端调用：deleteStockFromDb('159530', 'sz')
   ↓
3. 发送请求：{stock_code: '159530', stock_market: 'sz', type: 1}  ❌
   ↓
4. 后端 SQL：DELETE WHERE type = 1  ❌
   ↓
5. 实际数据：type = 3  ❌（不匹配）
   ↓
6. 删除 0 行，但返回 success: true  ❌
   ↓
7. 前端显示"已移除"，数据库数据还在  ❌
```

#### 数据库验证

**删除前**：
```sql
SELECT stock_code, type FROM custom_stocks WHERE user_id='admin' AND type=3;
-- 159530|3
-- 159816|3
-- 399283|3
-- 600031|3
```

**删除请求**：
```bash
curl -X POST /api/custom-stocks/delete \
  -d '{"stock_code":"159530","stock_market":"sz","type":1}'  # ❌ type=1
# 返回：{"success": true, "message": "已删除"}
```

**删除后**：
```sql
SELECT stock_code, type FROM custom_stocks WHERE user_id='admin' AND type=3;
-- 159530|3  ← 还在！❌
-- 159816|3
-- 399283|3
-- 600031|3
```

### 修复方案

**custom.js 修改**：
```javascript
async function deleteStockFromDb(code, market) {
  // 使用当前筛选的 type 值（删除当前分组的股票）
  const currentType = (typeof GroupManager !== 'undefined' && GroupManager.currentFilter) 
    ? GroupManager.currentFilter 
    : 1;
  console.log('📦 删除股票，使用 type:', currentType);
  
  const response = await fetch('/api/custom-stocks/delete', {
    body: JSON.stringify({ 
      stock_code: code, 
      stock_market: market, 
      type: currentType  // ✅ 使用当前分组的 type
    })
  });
}
```

**逻辑说明**：
- `GroupManager.currentFilter`：当前筛选的分组 type 值
- 如果 GroupManager 不存在或未定义，默认 type=1
- 删除时使用当前分组的 type，确保匹配正确的数据
