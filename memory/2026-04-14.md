## 2026-04-14 05:00 UTC - 修复分时图数据不完整问题

### 用户反馈

**问题**：分时图显示不对，5 条数据之前的数据应该用历史数据展示。

**示例接口**：
```
https://api.mairuiapi.com/hsstock/history/股票.SZ/5/n/FB1A859B-6832-4F70-AAA2-38274F23FC90?st=20260413&et=20260414&lt=300
```

### 问题分析

#### 原逻辑缺陷

**交易时间只获取最新 5 条数据**：
```javascript
// ❌ 修复前
if (isTradingTime) {
  // 只获取最新 5 条 1 分钟线
  const realtimeUrl = `.../latest/...?lt=5`;
}
```

**问题**：
- 10:00 时，只返回 09:55-10:00 的 5 条数据
- 09:30-09:55 的数据缺失 ❌
- 分时图前面大部分是空白 ❌

#### 用户需求

- 5 条实时数据之前的时间段，用历史数据填充
- 完整展示从开盘到当前的所有数据

### 修复方案

#### 新逻辑：历史数据 + 实时数据组合


### 数据源说明

**历史数据 API**：
```
https://api.mairuiapi.com/hsstock/history/{code}.{MARKET}/5/n/{LICENCE}?st={today}&et={today}&lt=300
```
- `5`: 5 分钟 K 线
- `lt=300`: 300 条数据（覆盖全天）
- 用于填充开盘到当前的完整时间轴

**实时数据 API**：
```
https://api.mairuiapi.com/hsstock/latest/{code}.{MARKET}/1/n/{LICENCE}?lt=5
```
- `1`: 1 分钟 K 线
- `lt=5`: 最新 5 条
- 用于更新最新时间点的精确数据
