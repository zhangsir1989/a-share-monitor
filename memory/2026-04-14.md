## 2026-04-14 06:00 UTC - 修复分时图昨收价错误导致涨幅计算不准确

### 用户反馈

**问题**：分时图显示的涨幅与实际不符，实际涨幅 2% 多，但图表显示只有约 0.7%。

### 问题分析

#### 根本原因

**MyData API 返回的 `pc` (prevClose) 字段含义不同**：

| API 类型 | pc 字段含义 |
|----------|------------|
| 历史数据 API | 第一个点的 pc = **昨收价** ✅ |
| 实时数据 API | 每个点的 pc = **前一根 K 线的收盘价** ❌ |

**实时数据示例**：
```
14:03: pc=70.51  ← 14:02 的收盘价
14:04: pc=70.56  ← 14:03 的收盘价
14:05: pc=70.52  ← 14:04 的收盘价
14:06: pc=70.63  ← 14:05 的收盘价
```

#### 原代码逻辑错误

```javascript
// ❌ 错误：用实时数据的 pc 覆盖历史数据的 pc
if (realtimeData[0]?.pc) prevClose = realtimeData[0].pc;
```

**问题流程**：
```
1. 获取历史数据
   ↓
   prevClose = historyData[0].pc = 70.02 ✅
   
2. 获取实时数据
   ↓
   
3. 用 realtimeData[0].pc = 70.52 覆盖 prevClose ❌
   ↓
   70.52 是前一根 K 线的收盘价，不是昨收价！
   
4. 涨幅计算错误
   (70.52 - 70.52) / 70.52 = 0% ❌
```

#### 正确数据

**历史数据第一个点**：
```json
{
  "t": "2026-04-14 09:35:00",
  "c": 69.15,
  "pc": 70.02  ← 这才是真正的昨收价！
}
```

### 修复方案

**删除覆盖 prevClose 的代码**，保持使用历史数据第一个点的 pc 作为昨收价。

```javascript
// ✅ 修复：不更新 prevClose
// 历史数据的第一个点的 pc 才是真正的昨收价
// 实时数据的 pc 是前一根 K 线的收盘价，不是昨收价
dataSource = 'mydata-history+realtime';
```

### 技术细节

#### MyData API pc 字段说明

**历史数据 API** (`/hsstock/history/...`):
- 第一个数据点的 `pc` = 昨收价
- 后续数据点的 `pc` = 前一根 K 线的收盘价

**实时数据 API** (`/hsstock/latest/...`):
- 所有数据点的 `pc` = 前一根 K 线的收盘价
- **不能**用实时数据的 pc 作为昨收价

#### 正确的昨收价获取方式

```javascript
// ✅ 正确：使用历史数据第一个点的 pc
prevClose = historyData[0]?.pc || 0;

// ❌ 错误：使用实时数据的 pc
prevClose = realtimeData[0]?.pc || 0;
```
