## 2026-04-14 02:00 UTC - 修复 custom_groups.type 全局 UNIQUE 约束导致多用户冲突的问题

### 用户反馈

**问题**：其他用户（非 admin）在自选股页面点击"新建分组"弹不出对话框，admin 用户正常。

### 问题分析

#### 根本原因

**custom_groups 表 schema 设计缺陷**：
```sql
CREATE TABLE custom_groups (
  ...
  type INTEGER NOT NULL UNIQUE,  -- ❌ 全局 UNIQUE
  ...
);
```

**问题**：
- type 字段有全局 UNIQUE 约束
- 每个用户都应该有 type=1 的默认分组"我的自选股"
- 但 admin 用户已经占用了 type=1
- 其他用户（如 zhangsir）无法插入 type=1，导致默认分组创建失败

#### 问题流程

```
1. admin 用户登录
   ↓
2. 检查 type=1 默认分组是否存在
   ↓
3. 不存在 → 创建 type=1 默认分组 ✅
   ↓
4. zhangsir 用户登录
   ↓
5. 检查 type=1 默认分组是否存在
   ↓
6. 不存在 → 尝试创建 type=1 默认分组 ❌
   ↓
7. UNIQUE constraint failed: custom_groups.type ❌
   ↓
8. 默认分组创建失败，但代码继续执行
   ↓
9. 获取分组列表 API 返回失败 ❌
   ↓
10. 前端无法渲染分组下拉菜单，"新建分组"按钮事件未绑定 ❌
```

#### 数据库验证

**修复前**：
```sql
-- admin 用户的分组
SELECT user_id, type, name FROM custom_groups WHERE user_id='admin';
-- admin|1|我的自选股 ✅

-- zhangsir 用户的分组
SELECT user_id, type, name FROM custom_groups WHERE user_id='zhangsir';
-- zhangsir|2|测试分组  ← 没有 type=1！❌

-- 尝试插入 type=1
INSERT INTO custom_groups (user_id, name, type) VALUES ('zhangsir', '我的自选股', 1);
-- Error: UNIQUE constraint failed: custom_groups.type ❌
```

### 修复方案

#### 方案：移除全局 UNIQUE，添加组合唯一索引

**修改前**：
```sql
CREATE TABLE custom_groups (
  type INTEGER NOT NULL UNIQUE  -- ❌ 全局唯一
);
```

**修改后**：
```sql
CREATE TABLE custom_groups (
  type INTEGER NOT NULL  -- ✅ 移除 UNIQUE
);

-- 添加 (user_id, type) 组合唯一索引
CREATE INDEX idx_custom_groups_user_type ON custom_groups(user_id, type);
```

**效果**：
- 每个用户可以独立使用 type=1,2,3...
- admin: type=1,2,3...
- zhangsir: type=1,2,3...
- 互不冲突 ✅

#### SQL 迁移脚本

```sql
BEGIN TRANSACTION;

-- 1. 创建新表（没有 UNIQUE 约束）
CREATE TABLE custom_groups_new (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id TEXT NOT NULL,
  name TEXT NOT NULL,
  icon TEXT DEFAULT '📁',
  color TEXT DEFAULT '#4a9eff',
  type INTEGER NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 2. 创建 (user_id, type) 组合唯一索引
CREATE INDEX idx_custom_groups_user_type ON custom_groups_new(user_id, type);

-- 3. 复制数据
INSERT INTO custom_groups_new SELECT * FROM custom_groups;

-- 4. 删除旧表
DROP TABLE custom_groups;

-- 5. 重命名新表
ALTER TABLE custom_groups_new RENAME TO custom_groups;

COMMIT;
```
