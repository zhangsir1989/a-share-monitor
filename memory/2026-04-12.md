### 问题修复 🔴

#### 1. 炸板个股涨跌幅字段错误

**问题**：使用了错误的 API 字段 `zdf` 而非 `zf`
- `zf`: 涨跌幅%（正确，如 8.83%）
- `zdf`: 其他涨幅指标（错误，如 1.32%）

**修复**：
```javascript
// 修复前
changePercent: item.zdf !== null ? item.zdf.toFixed(2) : '0.00'  // ❌ 1.32%

// 修复后
changePercent: item.zf !== null ? item.zf.toFixed(2) : '0.00'   // ✅ 8.83%
```

#### 2. 炸板个股连板数显示 None

**问题**：`lbc` 字段为 null 时显示为 JavaScript 的 `None` 字符串

**修复**：
```javascript
lbc: (item.lbc !== null && item.lbc !== undefined) ? item.lbc : 0
```

#### 3. 次新个股表格 colspan 错误

**问题**：表头 11 列，但加载状态 `colspan="10"` 只有 10 列

**修复**：`colspan="10"` → `colspan="11"`

## 炸板/次新分页显示修复 (2026-04-12 15:39-15:47 UTC)

### 用户反馈

炸板个股、次新个股前台只显示 10 条数据，分页显示"第 1 页 / 共 1 页"，但后台 Console 日志显示正确的分页（炸板 3 页、次新 13 页）。

### 问题根因

**分页 ID 转换缺失**：
- JS 代码使用驼峰命名：`breakBoardStocks`, `newBaseStocks`
- HTML ID 使用连字符：`break-board-current-page`, `new-base-current-page`
- `updatePagination()` 函数缺少这两种类型的 ID 转换
- 结果：`getElementById` 返回 `null`，分页更新失败，始终显示"第 1 页 / 共 1 页"

### 修复内容

**public/js/app.js** - `updatePagination()` 函数添加 ID 转换：
```javascript
function updatePagination(tableType, currentPage, totalPages) {
  let idPrefix;
  if (tableType === 'limitUpStocks') {
    idPrefix = 'limit-up';
  } else if (tableType === 'limitDownStocks') {
    idPrefix = 'limit-down';
  } else if (tableType === 'strongStocks') {
    idPrefix = 'strong';
  } else if (tableType === 'breakBoardStocks') {  // ✅ 新增
    idPrefix = 'break-board';
  } else if (tableType === 'newBaseStocks') {  // ✅ 新增
    idPrefix = 'new-base';
  } else {
    idPrefix = tableType;
  }
  // ...
}
```
