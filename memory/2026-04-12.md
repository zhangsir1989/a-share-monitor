## 系统错误修复 - 交易日计算 + 数据库自动创建 (2026-04-13 00:59-01:25 UTC)

### 用户反馈

"系统错误是什么意思，请修复，并且以后日志请暴露详细点"

### 问题分析

**错误日志**：
- `❌ 数据库初始化失败：Unexpected end of input` - 数据库文件损坏
- `获取涨停个股失败：Request failed with status code 404` - API 返回 404
- 炸板/次新个股数据显示 0 条

**根本原因**：

1. **时区计算错误**：
   - 系统时间已是 CST (UTC+8)
   - 代码中又加了 8 小时：`new Date(now.getTime() + (8 * 3600000))`
   - 导致北京时间变成 17:20，实际是 09:20
   - 周一 15:00 前的判断条件失效

2. **周一早盘数据不存在**：
   - 周一早上（15:00 前），今天的数据还没生成
   - MyData API 返回 404
   - 应该使用上周五（2026-04-10）的数据

3. **数据库文件损坏**：
   - users.db 文件损坏（145MB，异常大）
   - initDatabase() 直接读取文件，不存在就报错

4. **炸板/次新日期计算不统一**：
   - fetchBreakBoardStocks() 和 fetchNewBaseStocks() 有独立的日期计算逻辑
   - 没有使用统一的 getLatestTradeDate() 函数

### 修复方案

#### 1. 修复时区计算

**src/data-api.js** - getLatestTradeDate() 函数：
```javascript
// 修复前 - 错误的 +8 小时转换
const beijingTime = new Date(now.getTime() + (8 * 3600000));

// 修复后 - 系统时间已是 CST，不需要转换
const beijingTime = new Date();
```

#### 2. 周一早盘回退逻辑

```javascript
// 周一且 15:00 前，使用上周五（今天数据还没生成）
if (weekday === 1 && hour < 15) {
  const lastFriday = new Date(beijingTime);
  lastFriday.setDate(beijingTime.getDate() - 3);
  return lastFriday.toISOString().split('T')[0];
}

// 周日，使用周五
if (weekday === 0) {
  const friday = new Date(beijingTime);
  friday.setDate(beijingTime.getDate() - 2);
  return friday.toISOString().split('T')[0];
}

// 周六，使用周五
if (weekday === 6) {
  const friday = new Date(beijingTime);
  friday.setDate(beijingTime.getDate() - 1);
  return friday.toISOString().split('T')[0];
}
```

#### 3. 统一日期计算

**src/data-api.js** - fetchBreakBoardStocks() 和 fetchNewBaseStocks()：
```javascript
// 修复前 - 独立的日期计算
let dateStr;
if (tradeDate) {
  dateStr = tradeDate;
} else {
  const today = new Date();
  const weekday = today.getDay();
  // ... 复杂的日期计算
}

// 修复后 - 统一使用 getLatestTradeDate()
const dateStr = tradeDate || getLatestTradeDate();
```

#### 4. 数据库自动创建

**src/server.js** - initDatabase() 函数：
```javascript
async function initDatabase() {
  const SQL = await initSqlJs();
  
  // 如果数据库文件不存在，自动创建

```
