## 删除主侧边栏 X 按钮 + 修复 header 折叠按钮方向 (2026-04-12 05:49 UTC)

### 用户反馈
1. 主侧边栏 header 上的 X 按钮要删除（CSS 隐藏不够，需要物理删除）
2. 二级菜单 header 折叠按钮方向不对（图片显示 `》` 但应该是 `《`）

### 问题根因

**问题 1**: 主侧边栏 X 按钮
- HTML 中存在 `<button class="btn-close-sidebar" id="close-sidebar">✕</button>`
- 之前用 CSS 隐藏 (`display: none`) 但按钮还在 DOM 中
- 用户要求物理删除

**问题 2**: header 折叠按钮方向
- `initSubSidebarHeaderCollapse()` 创建按钮时硬编码为 `《`
- 但 `restoreSidebarState()` 可能已从 localStorage 恢复折叠状态 (`subCollapsed: true`)
- 导致视觉状态（`《`）与实际状态（折叠）不一致
- 应该根据实际状态设置图标

### 修复方案

#### 1. 删除主侧边栏 X 按钮 (HTML)

从以下文件中物理删除 X 按钮：
- `custom.html`
- `positions.html`
- `data-manager.html`
- `system.html`
- `users.html`

```html
<!-- 删除前 -->
<div class="sidebar-header">
  <h2>📊 系统导航</h2>
  <button class="btn-close-sidebar" id="close-sidebar">✕</button>
</div>

<!-- 删除后 -->
<div class="sidebar-header">
  <h2>📊 系统导航</h2>
</div>
```

#### 2. 修复 header 折叠按钮方向 (JS)

```javascript
// ✅ 修改后
function initSubSidebarHeaderCollapse() {
  const subSidebar = document.getElementById('sub-sidebar');
  if (!subSidebar) return;
  
  const header = subSidebar.querySelector('.sub-sidebar-header');
  if (!header) return;
  
  let collapseBtn = document.getElementById('sub-sidebar-header-collapse-btn');
  
  if (!collapseBtn) {
    collapseBtn = document.createElement('button');
    collapseBtn.id = 'sub-sidebar-header-collapse-btn';
    collapseBtn.className = 'sub-sidebar-header-collapse-btn';
    collapseBtn.title = '折叠菜单';
    header.appendChild(collapseBtn);
    console.log('✅ 二级侧边栏 header 折叠按钮已创建');
  }
  
  // ✅ 根据当前状态设置正确的图标
  updateSubSidebarHeaderButton(subSidebar);
  
  if (collapseBtn) {
    collapseBtn.addEventListener('click', handleSubSidebarHeaderCollapse);
  }
}
```

**图标逻辑**:
- **展开状态**: `《`（点击后折叠）
- **折叠状态**: `》`（点击后展开）
