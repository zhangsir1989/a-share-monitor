## 修复首页模块对齐 + 成交量单位转换 (2026-04-12 06:51 UTC)

### 用户反馈
1. 展开状态下模块未左右对齐（收缩状态对齐）
2. 成交金额展示不对

### 问题 1: 模块对齐问题

**根因分析**:
- `.container` 有 `padding: 20px`（通用内边距）
- `.header` 有 `padding-left: 0`（之前修复过）
- `.card-content` 有 `padding: 20px`（未修复）

**导致**:
```
Header 左侧：container padding-left + 20px + 0 = X
Card 左侧：  container padding-left + 20px + 20px = X + 20px ❌
```

**修复方案**:
```css
.card-content {
  padding: 20px;
  padding-left: 0;  /* 与 header 对齐，避免双重 padding */
}
```

### 问题 2: 成交量单位转换

**根因分析**:
- `data.totalVolume` 单位是**亿手**（API 返回股 ÷ 100000000）
- HTML 显示单位是**万手**
- 直接显示导致数值错误

**修复方案**:
```javascript
// 亿手转换为万手（1 亿手 = 10000 万手）
const totalVolumeWanShou = data.totalVolume * 10000;
elements.totalVolume.textContent = formatNumber(totalVolumeWanShou);
```

**示例**:
- 11.998 亿手 × 10000 = 119,980 万手 ✅
