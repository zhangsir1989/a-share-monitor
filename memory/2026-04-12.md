## 优化总成交金额计算逻辑 (2026-04-12 07:33 UTC)

### 用户反馈
- 总成交金额：先亿元计算相加，最后转换成万亿

### 优化方案

#### 1. 后端计算逻辑（data-api.js）

```javascript
// 转换为亿元（保留原始精度，不四舍五入）
const shAmount = shAmountYuan / 100000000;  // 沪市成交额（亿元）
const szAmount = szAmountYuan / 100000000;  // 深市成交额（亿元）
const totalAmount = shAmount + szAmount;  // 总计：亿元相加

return {
  totalAmount: Math.round(totalAmount * 100) / 100,  // 亿元，保留 2 位小数
  shAmount: Math.round(shAmount * 100) / 100,  // 亿元
  szAmount: Math.round(szAmount * 100) / 100,  // 亿元
};
```

#### 2. 前端显示逻辑（app.js）

```javascript
// 沪市成交额显示（单位：亿元，保留 2 位小数）
elements.shAmount.textContent = data.shAmount.toFixed(2);
elements.shAmountUnit.textContent = '亿元';

// 深市成交额显示（单位：亿元，保留 2 位小数）
elements.szAmount.textContent = data.szAmount.toFixed(2);
elements.szAmountUnit.textContent = '亿元';

// 总成交额显示（≥10000 亿元显示为万亿元）
if (data.totalAmount >= 10000) {
  elements.totalAmountUnit.textContent = '万亿元';
} else {
  elements.totalAmountUnit.textContent = '亿元';
}
```

### 计算流程

```
元 → 亿元（相加）→ 前端显示（万亿/亿）
```
