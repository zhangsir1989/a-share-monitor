## 首页日期显示修复 + 刷新间隔优化 (2026-04-13 02:24 UTC)

**用户反馈**："首页的数据显示不对吧，获取到的数据不是最新的交易日数据吧"

**问题根因**：
- 后端 API 正确返回 `tradeDate: 2026-04-10`（上周五）
- 前端 `updateDateDisplays()` 函数是空的，没有更新日期显示
- 用户看到"2026 年 04 月 13 日"（今天），但数据是上周五的，造成混淆

**修复内容**：

### 1. 修复日期显示函数
```javascript
function updateDateDisplays(dateStr) {
  if (!dateStr) return;
  
  // 获取 5 个模块的日期显示元素
  const limitUpDate = document.getElementById('limit-up-date');
  const limitDownDate = document.getElementById('limit-down-date');
  const strongDate = document.getElementById('strong-date');
  const breakBoardDate = document.getElementById('break-board-date');
  const newBaseDate = document.getElementById('new-base-date');
  
  // 格式化日期：2026-04-10 → 2026 年 04 月 10 日
  const [year, month, day] = dateStr.split('-');
  const formattedDate = `${year}年${month}月${day}日`;
  
  // 更新所有模块的日期显示
  if (limitUpDate) limitUpDate.textContent = formattedDate;
  if (limitDownDate) limitDownDate.textContent = formattedDate;
  if (strongDate) strongDate.textContent = formattedDate;
  if (breakBoardDate) breakBoardDate.textContent = formattedDate;
  if (newBaseDate) newBaseDate.textContent = formattedDate;
}
```

### 2. 修改默认刷新间隔
- 原值：3 秒
- 新值：**1 秒**（用户要求）

## 交易日逻辑修复 - 显示今天实时数据 (2026-04-13 02:30 UTC)

**用户反馈**："数据还是显示不对，界面上显示了上周五的数据，今天是交易日，应该显示当天的数据"

**问题根因**：
- 我的逻辑错误：假设"周一 15:00 前今天数据还没生成"
- 正确逻辑：交易日从 9:15 集合竞价开始就有实时数据
- 导致：周一 10:22（交易时间）显示上周五数据（错误）

**修复内容**：

### 修改交易日计算逻辑（三个文件）

**修复前（错误）**：
```javascript
if (weekday === 1 && hour < 15) {
  return "上周五";  // ❌ 错误！
}
```

**修复后（正确）**：
```javascript
// 周一至周五（交易日）：使用今天
// 周六/周日：使用上周五
if (weekday === 0) {  // 周日
  return "上周五";
}
if (weekday === 6) {  // 周六
  return "上周五";
}
return "今天";  // 周一 - 周五
```

### 修改文件
1. `src/data-api.js` - 后端 API 交易日计算
2. `src/server.js` - 服务器交易日计算
3. `public/js/app.js` - 前端交易日计算
4. `public/index.html` - 版本号更新为 `v=2026041303`
