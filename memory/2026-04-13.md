
## 首页日期选择器移除 + 自动刷新修复 (2026-04-13 02:09 UTC)

**用户需求**：
- 去除首页的日期选择器（红圈部分）
- 其他模块在交易时间自动刷新
- 按当前交易日进行自动刷新

**修复内容**：

### 1. 移除日期选择器 UI
- `public/index.html`：移除沪深成交量卡片的 `date-control-group`、`trade-date` input、`query-btn`

### 2. 简化前端逻辑
- `public/js/app.js`：
  - `fetchData()`：移除日期选择器逻辑，始终使用 `/api/all`（后端自动判断交易日）
  - `initDatePickers()`：简化为初始化函数
  - `updateDateDisplays()`：保留但不再使用（各模块右上角自动显示日期）

### 3. 修复交易日计算（关键 bug）
- `src/server.js` - `getLatestTradeDate()`：
  - **问题**：系统时间已是 GMT+8，代码又加了 8 小时，导致 10:08 变成 18:08
  - **修复**：移除重复的时区转换，直接使用系统时间
  - **结果**：周一 10:08（<15:00）正确返回上周五（2026-04-10）

### 4. 自动刷新逻辑（已存在，无需修改）
- `isTradingTime()`：判断是否在交易时间（9:30-11:30, 13:00-15:00）
- `startTimer()`：交易时间按设定间隔自动调用 `fetchData()`
- 默认刷新间隔：3 秒（用户可在设置中调整）

**自测结果**：
| 测试项 | 结果 |
|--------|------|
| 日期选择器移除 | ✅ 确认移除 |
| 交易日计算 | ✅ 2026-04-10（上周五） |
| 涨停数据 | ✅ 59 条 |
| 跌停数据 | ✅ 8 条 |
| 强势数据 | ✅ 188 条 |
| 炸板数据 | ✅ 27 条 |
| 次新数据 | ✅ 122 条 |
| 自动刷新 | ✅ 交易时间 3 秒间隔 |

**行为规则**：
| 时间 | 数据来源 | 刷新行为 |
|------|----------|----------|
| 周一 00:00-15:00 | 上周五 API 数据 | 交易时间自动刷新 |
| 周一 15:00 后 | 今天 API 数据 | 交易时间自动刷新 |
| 周二 - 周五 | 今天 API 数据 | 交易时间自动刷新 |
| 周六/周日 | 上周五 API 数据 | 不自动刷新（非交易时间） |
