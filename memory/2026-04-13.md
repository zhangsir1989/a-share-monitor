## 前端日期选择器修复 (2026-04-13 02:00 UTC)

**问题**：主页日期选择器显示今天（2026-04-13），但数据是上周五（2026-04-10）的，用户误以为数据没刷新

**根因**：前后端日期计算逻辑不一致
- 后端：正确处理"周一 15:00 前使用上周五"
- 前端：只处理周六/周日，**漏掉了周一情况**

**修复**：统一前后端逻辑，在 `public/js/app.js` 的 `getLatestTradeDate()` 函数中添加：
```javascript
// 周一且 15:00 前，使用上周五
if (weekday === 1 && hour < 15) {
  tradingDate.setDate(now.getDate() - 3);
  return tradingDate.toISOString().split('T')[0];
}
```

**验证**：
- ✅ 周一 09:55（北京时间）→ 日期选择器显示 2026-04-10（上周五）
- ✅ 数据日期与选择器一致
- ✅ 用户不再困惑

**行为规则**：
| 时间 | 显示日期 | 数据来源 |
|------|----------|----------|
| 周一 00:00-15:00 | 上周五 | 上周五 API 数据 |
| 周一 15:00 后 | 今天 | 今天 API 数据 |
| 周二 - 周五 | 今天 | 今天 API 数据 |
| 周六 | 上周五 | 上周五 API 数据 |
| 周日 | 上周五 | 上周五 API 数据 |
