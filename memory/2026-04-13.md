## 2026-04-13 13:00 UTC - 修复自选股分组筛选问题

### 问题描述
用户在存储分组（type=3）添加股票后，在存储分组页面查询不出来，只能在默认自选股（type=1）页面看到所有股票。

### 根本原因
1. **custom.js** 中 `loadStocks()` 函数硬编码了 `?type=1`：
   ```javascript
   const response = await fetch('/api/custom-stocks/list?type=1');  // ❌ 硬编码
   ```

2. **group-manager.js** 中 `filterByGroup()` 只在前端筛选：
   ```javascript
   function filterByGroup(type) {
     GroupManager.currentFilter = type;
     updateStockList();  // ❌ 只在前端筛选已加载的数据
   }
   ```

3. **结果**：`pageState.stocks` 只包含 type=1 的股票，前端筛选无法找到 type=3 的股票

### 修复方案

#### 1. custom.js - loadStocks 支持 type 参数
```javascript
async function loadStocks(type = 1) {
  console.log('📦 开始从数据库加载自选股 (type=' + type + ')...');
  const response = await fetch(`/api/custom-stocks/list?type=${type}`);
  // ...
}
```

#### 2. group-manager.js - filterByGroup 重新加载数据
```javascript
async function filterByGroup(type) {
  const newType = type === 'all' ? 1 : parseInt(type);
  GroupManager.currentFilter = newType;
  
  console.log('📁 筛选分组:', groupName, '(type=' + GroupManager.currentFilter + ')');
  
  // 重新从数据库加载该分组的股票
  console.log('🔄 切换分组，重新加载股票数据...');
  if (typeof loadStocks === 'function') {
    await loadStocks(newType);
  }
}
```
