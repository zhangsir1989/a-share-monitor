## 2026-04-13 16:10 UTC - 修复删除股票时 type=0 被错误转换为 type=1 的问题

### 用户反馈

**问题**：删除持仓或自选股时，前端显示"已移除"，但数据库数据并未实际删除，刷新页面后股票仍然存在。

### 问题分析

#### 根本原因

**server.js 删除 API**：
```javascript
const { stock_code, stock_market, type } = req.body;
const stockType = type || 1;  // ❌ type=0 时变成 1！
```

**JavaScript 陷阱**：
```javascript
0 || 1      // 1 ❌（0 是 falsy 值）
null || 1   // 1 ✅
undefined || 1  // 1 ✅
```

#### 问题流程

```
1. 用户删除持仓股票 00600 (type=0)
   ↓
2. 前端发送：{stock_code: '00600', stock_market: 'hk', type: 0}
   ↓
3. 后端处理：stockType = 0 || 1 → 1  ❌
   ↓
4. 执行 SQL：DELETE FROM custom_stocks WHERE type = 1  ❌
   ↓
5. 实际数据：type = 0  ❌（不匹配）
   ↓
6. 删除 0 行记录，但返回 success: true  ❌
   ↓
7. 前端显示"已移除"，但数据库数据还在  ❌
```

#### 数据库验证

**删除前**：
```sql
SELECT stock_code, stock_market, type FROM custom_stocks;
-- 00600|hk|0  ← 持仓
-- 00600|sz|2  ← AI 算力
-- 159530|sz|3
```

**删除请求**：
```bash
curl -X POST /api/custom-stocks/delete \
  -d '{"stock_code":"00600","stock_market":"hk","type":0}'
# 返回：{"success": true, "message": "已删除"}
```

**删除后**：
```sql
SELECT stock_code, stock_market, type FROM custom_stocks;
-- 00600|hk|0  ← 还在！❌
-- 00600|sz|2
-- 159530|sz|3
```

### 修复方案

**server.js 修改**：
```javascript
// ❌ 修复前
const stockType = type || 1;

// ✅ 修复后
const stockType = (type !== undefined && type !== null) ? parseInt(type) : 1;
```

**逻辑说明**：
- `type !== undefined && type !== null`：检查 type 是否有值
- `parseInt(type)`：转换为整数（防止字符串）
- `: 1`：如果没有提供 type，默认为 1（自选股）

**正确处理**：
```javascript
type=0      → stockType = 0  ✅
type=1      → stockType = 1  ✅
type=2      → stockType = 2  ✅
type=null   → stockType = 1  ✅
type=undefined → stockType = 1  ✅
```
