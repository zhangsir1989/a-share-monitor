## 2026-04-13 15:45 UTC - 持仓页面使用 type=0 并修复后端 type=0 处理逻辑

### 用户反馈

**问题**：在持仓页面添加股票时，股票被分配到"AI 算力"分组（type=2），但应该是持仓类型（type=0）。

**截图证据**：
- 用户在持仓页面添加股票
- 添加后股票出现在 AI 算力分组
- 用户期望：持仓页面应该使用独立的 type=0

### 问题分析

#### 问题 1: 前端使用错误的 type 值

**positions.js 修复前**：
```javascript
async function saveStockToDb(code, market) {
  const response = await fetch('/api/custom-stocks/add', {
    body: JSON.stringify({ stock_code: code, stock_market: market, type: 2 })  // ❌ type=2
  });
}

async function loadStocks() {
  const response = await fetch('/api/custom-stocks/list?type=2');  // ❌ type=2
}
```

**问题**：硬编码 `type=2`（AI 算力分组），应该是 `type=0`（持仓）。

#### 问题 2: 后端 type=0 处理逻辑错误

**server.js 修复前**：
```javascript
const { stock_code, stock_market, type } = req.body;
const stockType = type || 1;  // ❌ type=0 时变成 1
```

**问题**：JavaScript 中 `0 || 1` 的结果是 `1`，因为 `0` 是 falsy 值！

**测试**：
```javascript
0 || 1      // 1 ❌
null || 1   // 1 ✅
undefined || 1  // 1 ✅
```

**导致的问题**：
```
1. 前端发送 type=0
   ↓
2. 后端：stockType = 0 || 1 → 1  ❌
   ↓
3. 股票被保存到 type=1（我的自选股）  ❌
   ↓
4. 查询 type=0 时找不到股票  ❌
```

### 修复方案

#### 修复 1: positions.js 使用 type=0

**saveStockToDb()**：
```javascript
async function saveStockToDb(code, market) {
  const response = await fetch('/api/custom-stocks/add', {
    body: JSON.stringify({ 
      stock_code: code, 
      stock_market: market, 
      type: 0  // ✅ 持仓类型
    })
  });
}
```

**deleteStockFromDb()**：
```javascript
async function deleteStockFromDb(code, market) {
  const response = await fetch('/api/custom-stocks/delete', {
    body: JSON.stringify({ 
      stock_code: code, 
      stock_market: market, 
      type: 0  // ✅ 持仓类型
    })
  });
}
```

**loadStocks()**：
```javascript
async function loadStocks() {
  console.log('📦 开始从数据库加载持仓 (type=0)...');
  const response = await fetch('/api/custom-stocks/list?type=0');  // ✅
}
```

#### 修复 2: server.js 正确处理 type=0

**修改前**：
```javascript
const stockType = type || 1;  // ❌ type=0 → 1
```

**修改后**：
```javascript
const stockType = (type !== undefined && type !== null) ? parseInt(type) : 1;
// ✅ type=0 → 0
// ✅ type=1 → 1
// ✅ type=undefined → 1
// ✅ type=null → 1
```

**逻辑说明**：
- `type !== undefined && type !== null`：检查 type 是否有值
- `parseInt(type)`：转换为整数（防止字符串）
- `: 1`：如果没有提供 type，默认为 1（自选股）

### Type 值定义
