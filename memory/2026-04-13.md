## 分时图无数据显示修复 (2026-04-13 06:53 UTC)

**用户反馈**：分时图显示"暂无分时数据"

**问题根因**：
- `loadIntraday()` 和 `loadBasicInfo()` 调用 `IntradayChart.render()` 时
- 传递的是 `result.data`（数据数组）
- 但 `render()` 期望的是完整的结果对象 `{ success, data, prevClose }`
- `render()` 检查 `responseData.data` 时为 `undefined`
- 导致显示"暂无分时数据"占位符

**错误流程**：
```javascript
// 错误的调用
IntradayChart.render(result.data);
// result.data = [{time: '14:49', price: 11.06, ...}, ...]  // 数据数组

// render() 内部检查
if (!responseData || !responseData.data || responseData.data.length === 0) {
  this.drawPlaceholder('暂无分时数据');  // ❌ 触发占位符
  return;
}
// responseData.data = undefined (因为传入的是数组，不是对象)
```

**修复内容**：

1. **loadIntraday() 修复**：
   ```javascript
   // 修复前（错误）：
   IntradayChart.render(result.data);
   
   // 修复后（正确）：
   IntradayChart.render(result);  // 传递完整对象 { success, data, prevClose }
   ```

2. **loadBasicInfo() 修复**：
   ```javascript
   // 修复前（错误）：
   IntradayChart.render(intradayResult.data);
   
   // 修复后（正确）：
   IntradayChart.render(intradayResult);
   ```

**文件修改**：
- `public/stock-detail/js/main.js`: 修复两处 `render()` 调用

- 非交易时间：仅首次加载，不启动定时器

```javascript
function startAutoRefresh() {

  if (Utils.isTradingTime()) {
    console.log('✅ [startAutoRefresh] 交易时间，启动定时器刷新');
    // 启动定时器...
  } else {
    console.log('⏸️ [startAutoRefresh] 非交易时间，仅显示数据不刷新');
  }
}
```

## 分时图数据源修复 - 非交易时间使用历史数据 (2026-04-13 09:00 UTC)

**用户反馈**：分时图还是不对，应该使用历史分时图，今天的历史分时数据，非交易时刻

**问题现象**：
- 非交易时间（15:14）访问个股详细页
- 分时图只显示最后 5 条数据（14:56-15:00）
- 前面时间段都是空白，没有完整的历史分时曲线

**问题根因**：
- 后端 `/api/intraday/:code` 只使用**实时数据 API**
- 实时 API (`/hsstock/latest/.../1/n/...?lt=5`) 只返回最新 5 条 1 分钟数据
- 非交易时间没有实时数据更新，所以只有收盘前最后 5 条

## 修复方案

### 后端数据源策略（server.js）

**交易时间判断**：周一 - 周五 9:15-11:30 或 13:00-15:00

| 时间段 | 数据源 | MyData API | 数据条数 | 数据间隔 |
|--------|--------|-----------|---------|---------|
| **交易时间** | 实时数据 | `/hsstock/latest/{code}/1/n/{licence}?lt=5` | 5 条 | 1 分钟 |
| **非交易时间** | 历史数据 | `/hsstock/history/{code}/5/n/{licence}?lt=48` | 48 条 → 插值为 232 条 | 5 分钟 → 1 分钟 |

### 5 分钟 K 线插值逻辑

```javascript
// 每个 5 分钟 K 线生成 5 个 1 分钟数据点
for (let j = 0; j < 5; j++) {
  const minute = baseMinute + j;
  
  // 跳过无效时间段
  if (minute >= 60) continue;  // 跨小时
  if (baseHour === 11 && minute > 30) continue;  // 11:31-11:59（午休）
  if (baseHour === 12) continue;  // 12:00-12:59（午休）
  if (baseHour === 15 && minute > 0) continue;  // 15:01-15:59（收盘后）
  
  // 价格相同，成交量/额均分
  minuteData.push({
    time: `${String(baseHour).padStart(2, '0')}:${String(minute).padStart(2, '0')}`,
    price: item.c,  // 收盘价
    volume: item.v / 5,  // 成交量均分
    amount: item.a / 5   // 成交额均分
  });
}
```

## 2026-04-13 09:40 UTC - 分组管理功能修复（股票自动加入自选股）

### 问题报告
用户反馈：**在分组里添加股票后，股票没有出现在自选股列表中**

### 问题根因
分组系统（`custom_groups` + `stock_group_relations`）与自选股系统（`custom_stocks`）是**两个独立的表**：
- 用户通过分组选择器将股票分配到分组
- 股票只写入 `stock_group_relations` 表
- **没有自动写入** `custom_stocks` 表
- 导致：股票在分组里，但不在自选股列表中

### 错误逻辑
```
用户操作：点击股票卡片的「📁 分组」→ 勾选分组
    ↓
原流程：只调用 assignStockToGroup(code, groupId)
    ↓
数据库：INSERT INTO stock_group_relations
    ↓
结果：股票在分组里，但不在 custom_stocks 表
    ↓
问题：自选股页面看不到这只股票
```

### 修复方案

#### 核心原则
```
分组 = 自选股的标签/分类
        ↓
只有自选股才能被分配到分组
        ↓
分配到分组时自动检查，不在自选股则自动添加
```

#### 后端修复（server.js）

**修改 `app.post('/api/custom-groups/assign')`**：

```javascript
app.post('/api/custom-groups/assign', authenticateToken, async (req, res) => {
  const { code, groupId } = req.body;
  const userId = req.user.userId;
  
  // 1. 自动判断市场
  let market = code.startsWith('6') ? 'sh' : 
               code.startsWith('3') || code.startsWith('0') ? 'sz' :
               code.startsWith('4') || code.startsWith('8') ? 'bj' : 'hk';
  
  // 2. 【新增】检查是否在自选股中
  const checkStmt = db.prepare('SELECT 1 FROM custom_stocks WHERE user_id = ? AND stock_code = ?');
  const exists = checkStmt.get(userId, code);
  
  // 3. 【新增】不在自选股则自动添加
  if (!exists) {
    const insertStmt = db.prepare(`
      INSERT INTO custom_stocks (user_id, stock_code, stock_market, type, added_at)
      VALUES (?, ?, ?, 1, CURRENT_TIMESTAMP)
```

## 2026-04-13 10:38 UTC - 分组管理功能重构（type 字段关联）

### 用户需求
用户要求重新设计分组逻辑：
1. **新建分组时生成新的 type 值**
2. **股票的 type 字段 = 所属分组的 type**
3. **一只股票只能属于一个分组**（通过 type 字段关联）
4. **默认自选股（type=1）是独立分组**，不显示其他分组的股票
5. **系统重启后正常工作**

### 原设计问题

**原设计**（多对多关系）：
```
custom_stocks 表：type 固定为 1（自选股）
stock_group_relations 表：股票 - 分组多对多关联
    ↓
问题：一只股票可以属于多个分组，无法通过 type 区分
```

**新设计**（一对一关系）：
```
custom_groups 表：每个分组有唯一 type 值
custom_stocks 表：type 字段 = 股票所属分组的 type
    ↓
优势：通过 type 字段直接关联，一只股票只能属于一个分组
```
