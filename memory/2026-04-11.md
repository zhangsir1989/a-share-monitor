# 2026-04-11 - A 股实时监控服务开发记录

## 分时图功能完成

### MyData API 对接
- 使用接口：`https://api.mairuiapi.com/hsstock/history/{code}.{market}/5/n/{licence}?st={date}&et={date}&lt=48`
- 获取 5 分钟 K 线数据（48 条），转换为分时图格式
- 实时数据接口：`hsstock/latest/{code}.{market}/1/n/{licence}`

### 数据库存储
- 表名：`intraday_data`
- 字段：id, stock_code, trade_date, time, price, open, high, low, volume, amount, prevClose, created_at
- 索引：idx_intraday_code_date (stock_code, trade_date)
- 逻辑：
  - 历史数据从数据库获取
  - 当前时间数据从 MyData API 获取并存储
  - 自动创建表（首次访问时）

### 分时图 currentIndex 计算
- 上午交易时间（9:30-11:30）：索引 0-120
- 午休时间（11:30-13:00）：停留在索引 120
- 下午交易时间（13:00-15:00）：索引 121-241
- 价格曲线只绘制到当前时间

## 分时图对接 MyData 5 分钟 K 线 API

### API 接口
- 端点：`https://api.mairuiapi.com/hsstock/history/{code}.{market}/5/n/{licence}`
- 参数：`st={开始日期}&et={结束日期}&lt=48`
- 返回：48 条 5 分钟 K 线数据（09:35-15:00）

### 后端修改
- 文件：`src/server.js` `/api/intraday/:code` 端点
- 逻辑：
  - 调用 `fetchIntradayHistory(code, market, today)` 获取 5 分钟 K 线
  - 如果是周末/节假日，自动向前查找最近交易日
  - 批量存储到 intraday_data 表（如果不存在）
  - 返回 48 个数据点给前端

### 数据格式
```javascript
{
  time: '09:35',
  price: 11.09,
  open: 11.10,
  high: 11.12,
  low: 11.08,
  volume: 123456,
  amount: 1371234.56,
  prevClose: 11.10
}
```

### 前端绘制
- 时间轴：显示 09:35, 09:40, ..., 15:00（每 5 分钟一个标签）
- 价格曲线：连接 48 个数据点
- 成交量柱状图：对应每个时间点

### 测试
- 000001 平安银行：48 条数据，昨收 11.1，收盘 11.09
- 周末自动获取最近交易日（2026-04-10）数据

## 个股详情页自动刷新策略修改

### 刷新策略

| 模块 | 刷新频率 | 说明 |
|------|---------|------|
| 分时走势 | 用户选择（默认 500ms） | 实时刷新 |
| 买卖五档 | 用户选择（默认 500ms） | 实时刷新 |
| 基本信息（名称、价格、涨幅） | 用户选择（默认 500ms） | 实时刷新 |
| 基本数据 | 固定 1 秒 | 置灰显示，带提示标签 |
| 成交明细 | 已删除 | - |
| 资金流向 | 不自动刷新 | - |
| 逐笔成交 | 手动加载 | 不自动刷新 |

### 代码修改

**main.js**:
- `startAutoRefresh()`: 分离快速刷新（分时/五档/价格）和基本数据刷新
- `loadBasicInfo()`: 只更新股票名称、价格、涨幅和分时图
- `loadBasicDataOnly()`: 新增函数，只更新基本数据表格（1 秒刷新）
- `updateBasicDataUI()`: 新增函数，添加置灰样式和"1 秒刷新"提示
- 删除 `loadTradeDetail()` 和 `loadCapitalFlow()` 函数

**index.html**:
- 删除成交明细模块（trade-detail-card）

**main.css**:
- 添加 `.basic-data-slow-refresh` 样式类
- 置灰效果：`opacity: 0.85`
- 添加"1 秒刷新"提示标签样式
