## 修复成交额单位转换错误 (2026-04-11 11:19 UTC)

### 用户反馈
成交额好像不对

### 问题根因
**MyData API 返回数据单位理解错误**

API 返回格式：
```json
{
  "t": "2026-04-10 00:00:00",
  "v": 546415541,      // 成交量（股）
  "a": 955664107213.6, // 成交额（亿元）❌ 之前误以为是元
  ...
}
```

**实际数据**:
- 上证指数：`a = 955664107213.6` → **9556.64 亿元**
- 深证成指：`a = 1367204589022.18` → **13672.05 亿元**

**错误代码**:
```javascript
const shAmount = shData.a || 0;  // 误以为是元
const totalAmount = (shAmount + szAmount) / 100000000;  // ❌ 除以 1 亿
```

导致数据缩小 1 亿倍：
- 9556.64 亿元 ÷ 100000000 = 0.00009556 亿元 ❌

### 修复方案

**正确理解 API 返回单位**:
- `v`: 成交量（股）→ 需要转换为亿手（÷100000000）
- `a`: 成交额（亿元）→ **已经是亿元，无需转换**

**修复代码**:
```javascript
// MyData API 返回格式：v=成交量（股），a=成交额（亿元）
const shVolume = shData.v || 0;  // 成交量（股）
const szVolume = szData.v || 0;
const shAmount = shData.a || 0;  // 成交额（亿元）✅
const szAmount = szData.a || 0;

const totalAmount = shAmount + szAmount;  // 亿元，直接相加 ✅
const totalVolume = (shVolume + szVolume) / 100000000;  // 股→亿手 ✅

return {
  totalAmount: Math.round(totalAmount * 100) / 100,  // 亿元
  shAmount: Math.round(shAmount * 100) / 100,  // 亿元
  szAmount: Math.round(szAmount * 100) / 100,  // 亿元
};
```
