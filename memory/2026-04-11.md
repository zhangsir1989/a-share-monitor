## 修复成交额单位转换（元→亿元） (2026-04-11 11:49 UTC)

### 用户反馈
界面显示 2.30 不对

### 问题根因
**MyData API 返回的 a 字段单位是元，不是亿元**

API 实际返回：
```json
{
  "a": 955664107213.6  // 单位是元，不是亿元
}
```

**正确换算**:
- 上证指数：955664107213.6 元 = **9556.64 亿元**
- 深证成指：1367204589022.18 元 = **13672.05 亿元**
- 总计：**23228.69 亿元**

**错误代码**:
```javascript
const shAmount = shData.a || 0;  // ❌ 误以为是亿元，实际是元
const totalAmount = shAmount + szAmount;  // 955664107213.6 + 1367204589022.18
return {
  totalAmount: Math.round(totalAmount * 100) / 100  // 2322868696235.78
};
```

前端 `formatAmount` 函数：
```javascript
if (Math.abs(num) >= 10000) {
  const value = (num / 10000).toFixed(2);  // 2322868696235.78 / 10000 = 232286869.62
  return value + '万亿';
}
```

导致显示混乱，最终显示为 **2.30** ❌

### 修复方案

**正确单位转换（元→亿元）**:
```javascript
// MyData API 返回格式：v=成交量（股），a=成交额（元）
const shAmountYuan = shData.a || 0;  // 成交额（元）
const szAmountYuan = szData.a || 0;  // 成交额（元）

// 转换为亿元
const shAmount = shAmountYuan / 100000000;  // 9556.64 亿元
const szAmount = szAmountYuan / 100000000;  // 13672.05 亿元
const totalAmount = shAmount + szAmount;    // 23228.69 亿元

return {
  totalAmount: Math.round(totalAmount * 100) / 100  // 23228.69
};
```

前端显示：
```javascript
if (Math.abs(num) >= 10000) {
  // 23228.69 < 10000，不进入此分支
}
// 直接返回：23228.69 亿
```
