## 侧边栏折叠状态恢复修复 (2026-04-11 09:45 UTC)

### 用户反馈
点击二级菜单后，左侧有一大片空白（红圈标记顶部 header 区域）

### 问题根因
**核心问题**: `restoreSidebarState()` 函数定义了但从未被调用

**代码分析**:
```javascript
// sidebar.js 中定义了恢复函数
function restoreSidebarState() {
  const mainCollapsed = localStorage.getItem('sidebar-main-collapsed') === '1';
  const subCollapsed = localStorage.getItem('sidebar-sub-collapsed') === '1';
  // ... 设置类名和状态
  updateLayoutClasses();
}

// 但从未在任何地方调用这个函数！
initSidebar() {
  // ❌ 没有调用 restoreSidebarState()
  // ...
}
```

**问题表现**:
1. 用户折叠二级侧边栏
2. 状态保存到 localStorage
3. 刷新页面后，`restoreSidebarState()` 不会被调用
4. `sidebarState.subCollapsed` 保持默认值 `false`
5. `updateLayoutClasses()` 不会给 container 添加 `sub-collapsed` 类
6. `padding-left` 保持 460px（应该 280px）
7. 左侧出现 180px 空白区域

### 修复方案

**在 initSidebar() 中调用 restoreSidebarState()**:
```javascript
function initSidebar() {
  const sidebar = document.getElementById('sidebar');
  const subSidebar = document.getElementById('sub-sidebar');
  
  if (!sidebar) {
    console.warn('⚠️ 侧边栏元素不存在');
    return;
  }
  
  // 恢复之前保存的侧边栏状态 ✅
  restoreSidebarState();
  
  // ... 其他初始化逻辑
}
```

### 文件修改
- `sidebar.js`: 添加 `restoreSidebarState()` 调用
- 所有 HTML 页面：更新 `sidebar.js` 版本号（v=20260416）

### 效果
