## 修复切换页面时二级侧边栏状态错误 (2026-04-11 10:08 UTC)

### 用户反馈
点击二级菜单（自定义）后，主内容区左侧出现大片空白

### 问题根因
**场景**: 用户从首页（无二级侧边栏）点击「自定义」菜单

**问题链**:
1. 首页可能保存了 `subCollapsed: true` 的状态（从其他页面切换过来时）
2. 新页面（custom.html）加载时 `restoreSidebarState()` 被调用
3. 直接从 localStorage 读取 `subCollapsed: true` 并应用
4. 二级侧边栏错误地处于折叠状态
5. `container` 的 `padding-left` 保持 460px（应该是 280px，因为刚展开）
6. 左侧出现大片空白区域

**核心问题**: `restoreSidebarState()` 没有判断当前页面是否有二级侧边栏，直接应用了保存的状态

### 修复方案

**在 `restoreSidebarState()` 函数中添加判断**:

```javascript
// 恢复二级侧边栏状态（仅当当前页面有二级侧边栏时）
if (subSidebar) {
  if (subCollapsed) {
    subSidebar.classList.add('collapsed');
    sidebarState.subCollapsed = true;
  }
  console.log('✅ 二级侧边栏状态恢复:', subCollapsed ? '折叠' : '展开');
} else {
  // 当前页面没有二级侧边栏，重置状态
  sidebarState.subCollapsed = false;
  console.log('ℹ️ 当前页面无二级侧边栏，重置状态');
}
```

**逻辑说明**:
1. **当前页面有二级侧边栏** (`subSidebar` 存在)
   - 应用保存的折叠状态
   - 更新按钮图标
   
2. **当前页面无二级侧边栏** (`subSidebar` 不存在)
   - 重置 `sidebarState.subCollapsed = false`
   - 不应用折叠状态
   - 避免状态污染
