## 优化二级菜单展开逻辑 (2026-04-11 10:33 UTC)

### 用户反馈
点击一级菜单，二级菜单显示不出来，被自动收缩了

### 问题根因
**场景**: 用户点击一级菜单（如"自定义"）进入有二级菜单的页面

**问题链**:
1. 用户之前在某页面折叠了二级侧边栏
2. 状态保存到 localStorage (`subCollapsed: true`)
3. 点击一级菜单跳转到新页面（如 `/custom`）
4. 页面加载时 `restoreSidebarState()` 被调用
5. 从 localStorage 读取 `subCollapsed: true` 并应用
6. 二级菜单保持折叠状态 ❌
7. 用户看不到二级菜单选项 ❌

### 优化方案

**检测页面加载方式**:

```javascript
const isNavigation = document.referrer && document.referrer !== window.location.href;

if (isNavigation) {
  // 从其他页面跳转过来，默认展开二级菜单
  subSidebar.classList.remove('collapsed');
  sidebarState.subCollapsed = false;
  console.log('✅ 从其他页面跳转，二级侧边栏默认展开');
} else {
  // 刷新页面，恢复用户之前的折叠偏好
  if (subCollapsed) {
    subSidebar.classList.add('collapsed');
    sidebarState.subCollapsed = true;
  }
  console.log('✅ 刷新页面，恢复二级侧边栏状态');
}
```

**逻辑说明**:

| 场景 | document.referrer | 行为 |
|------|------------------|------|
| 点击一级菜单跳转 | 存在且不同 | 默认展开二级菜单 ✅ |
| 浏览器后退/前进 | 存在且不同 | 默认展开二级菜单 ✅ |
| 刷新页面 (F5) | 相同或空 | 恢复用户折叠偏好 ✅ |
| 直接访问 URL | 空 | 恢复用户折叠偏好 ✅ |
