## 修复二级侧边栏初始化状态不一致 (2026-04-11 10:17 UTC)

### 用户反馈
点击二级菜单的折叠按钮后，主内容区左侧出现大片空白

### 问题根因
**代码冲突**: `initSubSidebarCollapse()` 强制展开侧边栏，但 `restoreSidebarState()` 可能已恢复折叠状态

**问题链**:
1. 用户之前折叠了二级侧边栏，状态保存到 localStorage (`subCollapsed: true`)
2. 页面加载时，`restoreSidebarState()` 被调用
   - 从 localStorage 读取 `subCollapsed: true`
   - 给 subSidebar 添加 `collapsed` 类
   - 调用 `updateLayoutClasses()` 设置 `padding-left: 280px`
3. 然后 `initSubSidebarCollapse()` 被调用
   - **强制移除** `collapsed` 类 ❌
   - 视觉上侧边栏展开
   - 但 `sidebarState.subCollapsed` 仍然是 `true` ❌
4. 用户点击折叠按钮
   - `handleSubSidebarCollapse()` 切换状态
   - `sidebarState.subCollapsed` 变为 `false`
   - `updateLayoutClasses()` 设置 `padding-left: 460px` ❌
   - 但侧边栏实际上是展开的，应该是 280px

**核心问题**: 初始化时视觉状态与实际状态不一致

### 修复方案

**移除 `initSubSidebarCollapse()` 中的强制展开代码**:

修改前：
```javascript
// 确保二级侧边栏初始状态为展开（不折叠）
subSidebar.classList.remove('collapsed');  // ❌ 强制展开
updateSubSidebarButton(subSidebar);
```

修改后：
```javascript
// 根据保存的状态设置初始状态（不强制展开）
// restoreSidebarState() 已经设置了正确的状态和类名
// 这里只需要更新按钮图标
updateSubSidebarButton(subSidebar);
console.log('✅ 二级侧边栏初始状态：', sidebarState.subCollapsed ? '折叠' : '展开');
```

### 文件修改
- `sidebar.js`: `initSubSidebarCollapse()` 函数修复

### 状态同步流程

```
页面加载
  ↓
restoreSidebarState()
  - 读取 localStorage
  - 设置 sidebarState.subCollapsed
  - 添加/移除 collapsed 类
  - 调用 updateLayoutClasses()
  ↓
initSubSidebarCollapse()
  - 创建折叠按钮
  - 绑定点击事件
  - 更新按钮图标 ✅
  - 不再强制展开 ✅
  ↓
状态一致，布局正确
```
